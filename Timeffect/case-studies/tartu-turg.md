# Tartu Turg Booking System

- **Client**: Tartu Turg (Estonian market / tourism tours)
- **Problem**: After announcing guided tours, inbox exploded with 95 booking emails in 2 days. WordPress-only workflow made scheduling chaotic and manual.
- **Solution**: Custom booking interface + dedicated backend/CMS for managing tours, availability, and confirmations.
- **Tech**: Custom frontend (screenshot in `../assets/tartu-turg/booking-ui.jpg`) + tailored admin layer on top of WordPress data.
- **Timeline**: Built in ~2 hours using Cursor + automation.
- **Investment**: €600 (underpriced).
- **Impact**: Eliminated manual email back-and-forth; centralized bookings in one system.
